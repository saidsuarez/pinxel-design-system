Componente

Botones

Los botones Pinxel son píldoras claras y táctiles. La jerarquía se limita a primario, secundario, terciario, acento e icon button. Cada botón debe tener estados completos: default, hover, active, focus, disabled y loading.

Jerarquía de botones

Estas son las únicas variantes base. No crear nuevos colores de botón sin una razón de producto muy clara.

Primario

La acción principal de un bloque o flujo. Usar una sola vez por grupo de decisión.

Secundario

Alternativa visible con borde. Acompaña al primario sin competir.

Terciario

Acción contextual de bajo énfasis. Se comporta como botón, pero lee como texto accionable.

Acento

Acción comercial puntual. No reemplaza al primario en confirmaciones críticas.

Icon button

Herramienta compacta. Siempre debe tener aria-label.

Tamaños

Small solo para superficies densas; large para acciones de alta importancia.

Anatomía

  • Altura base: 44px; grande: 52px.
  • Altura pequeña: 36px, solo para barras densas o tablas.
  • Radio: --px-radius-pill.
  • Peso: 700; tamaño: 15px.
  • Iconos de 18px cuando acompañan texto.
  • Área táctil mínima: 44x44px para acciones principales e icon buttons.

Uso

  • Primario: guardar, crear, confirmar, continuar.
  • Secundario: cancelar, guardar borrador, revisar.
  • Terciario: ver detalles, saltar, editar luego, abrir ayuda.
  • Acento naranja: momentos comerciales puntuales.
  • Turquesa no se usa como variante de botón; se reserva para progreso, badges y estados positivos.
  • Icon button: herramientas compactas con aria-label.
  • Loading: bloquear doble envío con aria-busy="true".
  • Disabled: usar solo si la razón es evidente o se explica cerca del control.

Código base

<button class="px-button px-button--secondary" type="button">Guardar borrador</button>

<button class="px-button px-button--primary" type="button" aria-busy="true">Guardando</button>
